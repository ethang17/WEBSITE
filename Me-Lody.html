THE PROJECT

The Pitch

Our product is similar to the app BeReal. Instead of posting a picture once a day, our app is related to music. Once you download the app, the user would connect to their preferred music streaming platform. At a random time each day, a notification will be sent out informing the user that it's time to share your music with your friends. If you are currently listening to a song, the app will prompt the user if it is okay to post their song for those they have mutually added to see. If the user is not listening to music, the app will instead ask if it is okay to post their last listened to song.

My Contributions

I worked primarily on the front end of the app. One concept I implemented in the app is object oriented programming. This was used in various places of the app. For one, when creating the actual post of the app. information from the text inputs are used as paramters for an instance of what we called a "post". This object held a couple pieces of information such as the poster's username, the song information, the time of the post, as well as the associated picture. This information is then used in a function called card() which uses the information and formats it in a React view which is then returned and shown on the main page. I also did something similar with our friend system in which each account was its own instance of the Friend class and had a funciton in order to make the card which is displayed on the screen.

Plans For The App

We have no plans to continue with this app. If I was to try to create this app again I would likely start from scratch. I do not think that the coding that was done on it is of high enough quality to accomplish what we wanted to. I also think that the language was a big contributor to it being hard for us. Javascript may have been a better base for us to use than Tyepscript. This choice was made as the navigation was already created in a Typescript template.
